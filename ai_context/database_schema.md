# Local Database Schema (Isar)

This document defines the local data persistence model using Isar Database.
The application operates entirely client-side, making Isar the single source of truth for the user's portfolio.

## 1. Entities and Relationships

We start with a simple 1-to-N relationship between an `Isin` (the actual asset the user owns) and its `Ticker`s (the market-specific symbols used to fetch price data from Yahoo Finance).

### Entity: `Isin`
Represents a unique financial asset in the user's portfolio.

```dart
@collection
class Isin {
  Id id = Isar.autoIncrement;

  @Index(unique: true)
  late String isinCode; // e.g., "US0378331005"

  late String name; // e.g., "Apple Inc."

  // The actual position/amount invested by the user
  double position = 0.0; 
  
  // The price at which the asset was purchased
  double purchasePrice = 0.0;
  
  // Currency in which the asset was purchased (e.g. USD, EUR)
  late String currency;
  
  // 1 to N relationship: One ISIN can be tracked via multiple markets
  final tickers = IsarLinks<Ticker>(); 
}
```

### Entity: `Ticker`
Represents a specific market symbol for an ISIN. Used to fetch data from Yahoo Finance.

```dart
@collection
class Ticker {
  Id id = Isar.autoIncrement;
  
  @Index(unique: true)
  late String symbol; // e.g., "AAPL" (NASDAQ) or "APC.F" (Frankfurt)
  
  late String exchange; // e.g., "NASDAQ" or "LSE"
  
  // Currency in which this ticker trades (e.g., "USD", "EUR", "GBP")
  late String currency;
  
  // Backlink to the parent Isin
  @Backlink(to: 'tickers')
  final isin = IsarLink<Isin>();
}
```

## 3. Caching Entities

### Entity: `MarketDataCache`
Designed to cache Yahoo Finance data, avoiding network saturation and providing offline capabilities for the **Markets Tab**. Contains Level 1 (current quote) and Level 2 (intraday sparkline) data.

```dart
@collection
class MarketDataCache {
  Id id = Isar.autoIncrement;

  // The symbol this cache belongs to (e.g. "GOOGL")
  @Index(unique: true)
  late String symbol;

  // Timestamp of the last successful fetch
  late DateTime lastUpdated;

  // Level 1 Data
  double regularMarketPrice = 0.0;
  double chartPreviousClose = 0.0;
  
  // Level 2 Data (Intraday Chart points - usually around 80 data points for a single day at 5m interval)
  List<double> intradayPrices = [];

  // Backlink to the parent Ticker
  @Backlink(to: 'marketDataCache')
  final ticker = IsarLink<Ticker>();
}
```
*Note: Level 3 Data (Historical Charts) are not permanently cached in Isar, they are fetched on demand when opening the detail view.*

## 4. Future Evolutions
As the app grows, we will append these entities:

- `NewsCache`: To store offline summaries generated by the AI for specific ISINs.
